Splynek — Safari bookmarklets

Drag the buttons below onto Safari's Bookmarks Bar. Click one on any page to hand its URL to the Splynek macOS app.

1 — Download with Splynek

↓ Splynek

Drag this to the bookmarks bar. Click it while viewing any page whose URL you want to hand to Splynek. Safari prompts the first time to confirm opening the app.

2 — Add page to Splynek queue

＋ Splynek queue

Same as above, but appends to the persistent queue instead of starting immediately.

3 — Download the link under the cursor

↓ Splynek (link)

Click this with text selected or after tab-focusing a link; it picks up the nearest anchor's href. Falls back to a prompt.

Install

  1. Enable Safari's Bookmarks Bar: View → Show Favorites Bar (⌘⇧B).
  2. Drag any button above onto the bar.
  3. Rename it if you like (right-click → Rename).
  4. Click it on any page to invoke Splynek. Safari asks once per scheme; tick Always allow to silence the prompt permanently.

Why not a Safari App Extension?

A proper Safari extension would live in the Splynek app bundle as a .appex. That requires an Xcode project, which the current Swift-Package-Manager-only build target doesn't produce. Bookmarklets are zero-install, work everywhere, and cover the same 90% of use cases (one-click hand-off of the current page).

What gets sent

splynek://download?url=<percent-encoded>&start=1
splynek://queue?url=<percent-encoded>

Splynek treats these identically to drag-and-drop, App Intents, or the Chrome extension — same code path, same interface selection, same integrity checks.
